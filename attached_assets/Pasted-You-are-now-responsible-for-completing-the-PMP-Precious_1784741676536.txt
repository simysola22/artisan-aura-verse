PHASE 1 — AUTHENTICATION AND IDENTITY
==================================================

Verify the complete Clerk → PMP identity flow.

Required:

Provider:
- Google sign-up
- Email sign-up if supported
- Sign-in
- Sign-out
- Session restoration
- Correct provider account type

Employer:
- Google sign-up
- Email sign-up if supported
- Sign-in
- Sign-out
- Session restoration
- Correct employer account type

Flow:

1. User selects account type.
2. Clerk authentication begins.
3. User completes registration.
4. Clerk session becomes active.
5. Frontend obtains Clerk token.
6. Frontend calls GET /v1/auth/me.
7. If PMP identity does not exist, call POST /v1/auth/sync.
8. Backend verifies Clerk token.
9. Backend creates PMP identity with correct account type.
10. Frontend becomes authenticated.
11. User is routed to correct dashboard/onboarding.

Do not allow:
- User to appear authenticated when PMP sync failed.
- Provider to become employer accidentally.
- Employer to become provider accidentally.
- Google OAuth redirect to lose account type.
- Missing account type to silently default to the wrong role.

Use the strongest reliable account-type persistence mechanism available in the current Clerk setup.
Use Clerk metadata where appropriate, with safe fallback handling.

Add proper:
- loading state
- sync state
- error state
- retry state
- sign-out state

Test this manually.
