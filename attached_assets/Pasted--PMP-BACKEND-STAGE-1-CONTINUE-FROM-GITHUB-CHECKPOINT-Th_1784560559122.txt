# PHASE 4 — COMPLETE THE BACKEND FOUNDATION

Complete only the Backend Stage 1 foundation.

The following must be operational or clearly established:

## Configuration

* Centralized configuration
* Environment validation
* `.env.example`
* Clear failure for invalid required configuration
* No committed secrets

## HTTP/API

* Portable server
* Centralized application creation
* Versioned API strategy
* Request validation foundation
* Centralized error handling
* Consistent error format
* Request IDs

## Database

* PostgreSQL-compatible architecture
* Drizzle ORM or the already-selected database layer
* Version-controlled migrations
* Configurable database connection
* Connection pooling
* Database health/readiness integration

## Logging

* Structured logging
* Request context
* Request ID support
* Safe error logging
* No passwords, tokens, or secrets in logs

## Security

* CORS configuration
* Security headers
* Authentication middleware foundation
* Authorization middleware foundation
* Safe error responses
* Server-side validation foundation

## Rate limiting

Maintain a provider abstraction capable of supporting a distributed deployment.

An in-memory fallback may exist for local development only.

Do not represent in-memory rate limiting as sufficient for a multi-instance production deployment.

## Cache

Maintain a replaceable cache abstraction.

It must be possible to use an external Redis-compatible cache without rewriting business logic.

## Storage

Maintain a replaceable storage abstraction.

Local development storage must not become a production requirement.

The architecture must support external object storage.

## Email

Maintain a replaceable email abstraction.

Console email may be used for development only.

## Health

Implement and verify:

```text
GET /health
```

and, where appropriate:

```text
GET /ready
```

Clearly distinguish liveness from readiness.

## Graceful shutdown

The server must:

1. Stop accepting new requests.
2. Allow active requests to finish appropriately.
3. Close database connections.
4. Close external connections.
5. Stop workers where applicable.
6. Exit cleanly.
